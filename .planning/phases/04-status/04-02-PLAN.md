**Add to the process section, BEFORE the workflow execution call:**

```
**Azure DevOps Status Integration (automatic, conditional):**

Before executing the workflow, check if `.planning/azdev-task-map.json` exists in $CWD via Bash:
  `test -f .planning/azdev-task-map.json && echo "HAS_MAP" || echo "NO_MAP"`

If HAS_MAP:
1. Read `.planning/azdev-task-map.json` using the Read tool.
2. For each mapping entry, find the mapping whose `repoPath` matches $CWD (or if running from the PlanningMe repo, process all mappings).
3. For each relevant mapping:
   a. Mark the story as Active:
      `node ~/.claude/get-shit-done/bin/azdev-tools.cjs update-state --id {storyId} --state Active --cwd $CWD 2>/dev/null || true`
   b. Mark each task as Active:
      `node ~/.claude/get-shit-done/bin/azdev-tools.cjs update-state --id {taskId} --state Active --cwd $CWD 2>/dev/null || true`
4. Status update failures are non-blocking (`|| true`). If stderr output appears, note it but continue execution.

Then proceed with the normal workflow execution.
```

**Add to the process section, AFTER the workflow execution completes successfully:**

```
**Azure DevOps Status Completion (automatic, conditional):**

After the workflow completes successfully (all plans executed, verification passed):

If `.planning/azdev-task-map.json` exists:
1. Read the task map again.
2. For each relevant mapping:
   a. Mark each mapped task as Resolved:
      `node ~/.claude/get-shit-done/bin/azdev-tools.cjs update-state --id {taskId} --state Resolved --cwd $CWD 2>/dev/null || true`
   b. Check story children states:
      `node ~/.claude/get-shit-done/bin/azdev-tools.cjs get-child-states --id {storyId} --cwd $CWD`
   c. Parse the JSON result. If `allResolved` is true:
      - Mark the story as Resolved:
        `node ~/.claude/get-shit-done/bin/azdev-tools.cjs update-state --id {storyId} --state Resolved --cwd $CWD 2>/dev/null || true`
      - Print: "All tasks resolved. Story #{storyId} marked as Resolved."
   d. If `allResolved` is false:
      - Print notification:
        "Execution done. {N} tasks still open: [{list open task titles}]. Story #{storyId} stays Active until all resolved."
      - Do NOT attempt to resolve the story.
3. All status update failures are non-blocking. Execution result is not affected by state update errors.
```
