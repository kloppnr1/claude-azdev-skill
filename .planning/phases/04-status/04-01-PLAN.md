Add two new async functions and their CLI router entries to azdev-tools.cjs.

**cmdUpdateState(cwd, args):**
- Parse `--id` and `--state` from args (same pattern as cmdUpdateDescription).
- If either missing, stderr usage message + exit 1.
- loadConfig(cwd), encode PAT with `Buffer.from(':' + cfg.pat).toString('base64')`.
- PATCH URL: `${cfg.org}/${cfg.project}/_apis/wit/workitems/${id}?api-version=7.1`
- Patch body: `[{ op: 'add', path: '/fields/System.State', value: state }]` — use `op: 'add'` (NOT 'replace') per Azure DevOps API convention for System.State.
- Call `makePatchRequest(patchUrl, encodedPat, patchBody)`.
- On status 200: stdout `JSON.stringify({ status: 'updated', id: Number(id), state })`, exit 0.
- On status 400 (invalid transition): stderr "Invalid state transition for work item {id}: {errorBody.message}. Check that '{state}' is a valid target state.", exit 1.
- On status 403: stderr "Status update failed for work item {id}: PAT needs vso.work_write scope. Regenerate at https://dev.azure.com/_usersSettings/tokens and re-run /gsd:azdev-setup.", exit 1.
- On other errors: stderr error message, exit 1.

**cmdGetChildStates(cwd, args):**
- Parse `--id` from args.
- If missing, stderr usage message + exit 1.
- loadConfig(cwd), encode PAT.
- Step 1: GET `${cfg.org}/${cfg.project}/_apis/wit/workitems/${id}?$expand=relations&api-version=7.1` via makeRequest.
- Step 2: Filter relations to `rel === 'System.LinkTypes.Hierarchy-Forward'`, extract child IDs from URL (last path segment).
- Step 3: If no children found, stdout `JSON.stringify({ allResolved: true, children: [] })`, exit 0.
- Step 4: Batch GET child states via POST to `${cfg.org}/${cfg.project}/_apis/wit/workitemsbatch?api-version=7.1` with body `{ ids: childIds, fields: ['System.Id', 'System.Title', 'System.State'], errorPolicy: 'omit' }`. Use `makeRequest(batchUrl, encodedPat, 'POST', batchBody)`.
- Step 5: Parse response, map to `[{ id, title, state }]` from `fields['System.Id']`, `fields['System.Title']`, `fields['System.State']`.
- Step 6: Determine `allResolved = children.every(c => c.state === 'Resolved' || c.state === 'Closed' || c.state === 'Done')` — accept Resolved, Closed, and Done as "completed" states to handle different process templates.
- stdout `JSON.stringify({ allResolved, children })`, exit 0 always (caller decides).
