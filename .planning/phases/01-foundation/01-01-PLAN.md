  <name>Task 1: Create azdo-tools.cjs with config management and HTTP client</name>
  <files>~/.claude/get-shit-done/bin/azdo-tools.cjs</files>
  <action>
Create `~/.claude/get-shit-done/bin/azdo-tools.cjs` as a CommonJS Node.js script following the `gsd-tools.cjs` pattern. The script must use ONLY Node.js built-ins (fs, path, https, Buffer). No external dependencies.

Structure:
1. Shebang: `#!/usr/bin/env node`
2. Require: `fs`, `path`, `https`
3. Config helpers section:
   - `getConfigPath(cwd)` -- returns `path.join(cwd, '.planning', 'azdo-config.json')`
   - `loadConfig(cwd)` -- reads and parses config, decodes PAT (strip base64, remove leading colon). Returns `{org, project, pat}` with raw PAT. Throws descriptive error if file missing.
   - `saveConfig(cwd, {org, project, pat})` -- normalizes org URL (strip `https://dev.azure.com/` prefix and trailing slashes, store only slug). Encodes PAT as `Buffer.from(':' + pat).toString('base64')`. Writes JSON with 2-space indent.
   - `normaliseOrg(input)` -- strips `https://dev.azure.com/` prefix, any trailing slashes, returns org slug only. Also handles `https://{server}/{collection}` by stripping protocol+host.

4. HTTP helper section:
   - `makeRequest(url, encodedPat, method='GET', body=null)` -- returns Promise resolving to `{status, body}`. Uses `https.request` with `Authorization: Basic ${encodedPat}`, `Content-Type: application/json`, `Accept: application/json`. Handles both GET and PATCH (for future phases). Rejects on network error with descriptive message.

5. CLI commands section:
   - `cmdSaveConfig(cwd, args)` -- parses `--org`, `--project`, `--pat` from args. Validates all three present. Calls `saveConfig`. Outputs JSON `{"status":"saved","org":"...","project":"..."}` to stdout. Exit 0.
   - `cmdLoadConfig(cwd)` -- calls `loadConfig`. Outputs JSON `{"org":"...","project":"...","pat":"<raw>"}` to stdout. Exit 0. If no config, stderr error + exit 1.
   - `cmdTest(cwd)` -- loads config, decodes PAT, encodes for auth header. Makes TWO requests:
     a. `GET https://dev.azure.com/${org}/_apis/projects?$top=1&api-version=7.1` -- verifies auth + vso.project scope
     b. If (a) succeeds (200): `GET https://dev.azure.com/${org}/${project}/_apis/wit/workitems?ids=1&api-version=7.1` -- verifies vso.work scope. A 404 is acceptable (item not found = auth OK). Only 401/403 means scope missing.
     Success output: `"Connected to ${org}/${project}"` to stdout, exit 0.
     Auth failures:
       - 401 on projects: stderr `"Authentication failed. Check your PAT is correct and has not expired. Run /gsd:azdo-setup to reconfigure."` + exit 1
       - 403 on projects: stderr `"Authorisation denied. Check your PAT has the vso.project scope. Run /gsd:azdo-setup to reconfigure."` + exit 1
       - 401/403 on work items: stderr `"Authentication succeeded but work items access denied. Check your PAT has the vso.work scope. Run /gsd:azdo-setup to reconfigure."` + exit 1
       - Other status: stderr `"Unexpected response: HTTP ${status}"` + exit 1

6. CLI router (main function):
   - Parse `process.argv.slice(2)`, extract command and `--cwd` flag (default `process.cwd()`)
   - Route: `save-config` -> `cmdSaveConfig`, `load-config` -> `cmdLoadConfig`, `test` -> `cmdTest`
   - Unknown command: stderr usage message + exit 1
   - Top-level `.catch()` on main: stderr error.message + exit 1

Key implementation notes:
- PAT encoding: `Buffer.from(':' + pat).toString('base64')` (colon prefix is mandatory, empty username)
- PAT decoding: `Buffer.from(encoded, 'base64').toString('utf-8').slice(1)` (remove leading colon)
- Org normalization: handle `https://dev.azure.com/myorg/`, `https://dev.azure.com/myorg`, and `myorg` -- all produce `myorg`
- Use `process.exit(0)` on success, `process.exit(1)` on any failure
- All output to stdout is JSON or single-line text (parseable by Bash)
- All errors to stderr via `console.error()`
